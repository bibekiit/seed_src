PURPOSE

FREQEST - Estimate fingerprint ridge frequency within image block

SYNOPSIS

function freqim = freqest(im, orientim, windsze, minWaveLength, maxWaveLength)

DESCRIPTION 

 FREQEST - Estimate fingerprint ridge frequency within image block

 Function to estimate the fingerprint ridge frequency within a small block
 of a fingerprint image.  This function is used by RIDGEFREQ

 Usage:
  freqim =  freqest(im, orientim, windsze, minWaveLength, maxWaveLength)

 Arguments:
         im       - Image block to be processed.
         orientim - Ridge orientation image of image block.
         windsze  - Window length used to identify peaks. This should be
                    an odd integer, say 3 or 5.
         minWaveLength,  maxWaveLength - Minimum and maximum ridge
                     wavelengths, in pixels, considered acceptable.
 
 Returns:
         freqim    - An image block the same size as im with all values
                     set to the estimated ridge spatial frequency.  If a
                     ridge frequency cannot be found, or cannot be found
                     within the limits set by min and max Wavlength
                     freqim is set to zeros.

 Suggested parameters for a 500dpi fingerprint image
   freqim = freqest(im,orientim, 5, 5, 15);

 See also:  RIDGEFREQ, RIDGEORIENT, RIDGESEGMENT

 Note I am not entirely satisfied with the output of this function.

0041     
0042     debug = 0;
0043     
0044     [rows,cols] = size(im);
0045     
0046     % Find mean orientation within the block. This is done by averaging the
0047     % sines and cosines of the doubled angles before reconstructing the
0048     % angle again.  This avoids wraparound problems at the origin.
0049     orientim = 2*orientim(:);    
0050     cosorient = mean(cos(orientim));
0051     sinorient = mean(sin(orientim));    
0052     orient = atan2(sinorient,cosorient)/2;
0053 
0054     % Rotate the image block so that the ridges are vertical
0055     rotim = imrotate(im,orient/pi*180+90,'nearest', 'crop');
0056     
0057     % Now crop the image so that the rotated image does not contain any
0058     % invalid regions.  This prevents the projection down the columns
0059     % from being mucked up.
0060     cropsze = fix(rows/sqrt(2)); offset = fix((rows-cropsze)/2);
0061     rotim = rotim(offset:offset+cropsze, offset:offset+cropsze);
0062 
0063     % Sum down the columns to get a projection of the grey values down
0064     % the ridges.
0065     proj = sum(rotim);
0066     
0067     % Find peaks in projected grey values by performing a greyscale
0068     % dilation and then finding where the dilation equals the original
0069     % values.
0070     dilation = ordfilt2(proj, windsze, ones(1,windsze));
0071     maxpts = (dilation == proj) & (proj > mean(proj));
0072     maxind = find(maxpts);
0073 
0074     % Determine the spatial frequency of the ridges by divinding the
0075     % distance between the 1st and last peaks by the (No of peaks-1). If no
0076     % peaks are detected, or the wavelength is outside the allowed bounds,
0077     % the frequency image is set to 0
0078     if length(maxind) < 2
0079     freqim = zeros(size(im));
0080     else
0081     NoOfPeaks = length(maxind);
0082     waveLength = (maxind(end)-maxind(1))/(NoOfPeaks-1);
0083     if waveLength > minWaveLength & waveLength < maxWaveLength
0084         freqim = 1/waveLength * ones(size(im));
